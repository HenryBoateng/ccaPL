The CCA editor has three different output types for the code, being execution output, communication graph and behaviour graph. Try these examples below to see the differences.

The Execution Output

This is the default output known as the execution output. This essentially outputs the */ //The symbol ‘< -- >’ corresponds to the structural congruence relation. ‘-->’ represents the reduction relation The notation ‘{ root ===(hello)===> root}’ means that an ambient ‘root’ has sent a message ‘hello’ to another ambient ‘root’ Other notations such as ‘Child to parent’ and ‘Sibling to sibling’ provide information about the relationship between the sender ‘A’ and the receiver ‘B’.

The Communication Graph

The top row in the diagram lists the names of the ambients involved in the system being executed. The vertical dashed line represents the timeline for each ambients (time increases from top to bottom). A solid directed line from the timeline of an ambient to that of another ambient (or in this case itself) indicates a message passing from one ambient to another at a specific time point, with the content of the message carried as a label to that line.

The Behaviour Graph

The top row in the diagram lists the names of the ambients involved in the system being executed. The vertical dashed line represents the timeline for each ambients (time increases from top to bottom). A solid directed line from the timeline of an ambient to that of another ambient (or in this case itself) indicates a message passing from one ambient to another at a specific time point, with the content of the message carried as a label to that line. A behaviour graph provides additional information about the mobility of ambients. A box labelled as “X - -> Y” on the timeline of an ambient B indicates that the ambient B has moved from location X to location Y.
